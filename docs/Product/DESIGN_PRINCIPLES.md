# Design Principles — Evol OS

## Objetivo

Este documento define os princípios obrigatórios para qualquer tela do Evol OS.

Não são sugestões.

São padrões que garantem consistência, simplicidade e uma excelente experiência para todos os usuários.

---

# 1. Hierarquia Visual

Toda tela deve responder imediatamente:

1. Onde estou?
2. O que posso fazer?
3. Qual é a ação principal?

Nunca apresentar informações sem hierarquia.

---

# 2. Um objetivo por tela

Cada tela possui uma missão principal.

Evite telas tentando resolver muitos problemas ao mesmo tempo.

---

# 3. CTA Principal

Toda página possui UMA ação principal.

Exemplos:

Criar colaborador

Criar avaliação

Enviar feedback

Criar PDI

Esse botão deve possuir maior destaque visual.

---

# 4. Estados Vazios

Nunca mostrar apenas:

"Nenhum registro encontrado."

Sempre ensinar.

Exemplo:

Você ainda não possui avaliações.

As avaliações ajudam gestores a acompanhar o desenvolvimento das equipes.

[ Criar primeira avaliação ]

---

# 5. Ajuda Contextual

Sempre que existir um conceito novo:

ⓘ

Explicar em linguagem simples.

Nunca exigir consulta ao manual.

---

# 6. Fluxos Guiados

Sempre que existir um processo com várias etapas:

Mostrar progresso.

Exemplo:

Etapa 1 de 4

Criar template

↓

Criar seção

↓

Criar perguntas

↓

Criar ciclo

---

# 7. Feedback Imediato

Toda ação gera retorno.

✔ Salvo

⏳ Salvando...

⚠ Erro

🎉 Concluído

---

# 8. Próximo Passo

Nenhuma ação termina sem indicar o próximo passo.

Exemplo:

Departamento criado.

Agora cadastre os cargos.

[ Criar cargo ]

---

# 9. Coach Mode

Sempre que possível:

Mostrar dicas curtas.

Boas práticas.

Erros comuns.

Recomendações.

Sem interromper o fluxo.

---

# 10. Progressive Disclosure

Mostrar apenas o necessário.

Recursos avançados aparecem conforme o usuário evolui.

---

# 11. Linguagem

Escrever como pessoas falam.

Evitar jargões.

Preferir:

Modelo de Avaliação

ao invés de

Template

---

# 12. Ícones

Ícones nunca substituem texto.

Eles apenas reforçam significado.

---

# 13. Cores

Verde

Sucesso

Azul

Informação

Amarelo

Atenção

Vermelho

Erro

Nunca utilizar cores como único indicador.

Sempre combinar com texto.

---

# 14. Formulários

Explicar campos importantes.

Agrupar informações relacionadas.

Evitar formulários muito longos.

Sempre informar campos obrigatórios.

---

# 15. Tabelas

Mostrar apenas informações importantes.

Ações principais sempre visíveis.

Busca sempre disponível quando fizer sentido.

Filtros simples.

---

# 16. Dashboards

Responder:

O que aconteceu?

Por que aconteceu?

O que devo fazer agora?

---

# 17. IA

A IA nunca invade a experiência.

Ela aparece como auxílio.

Sempre opcional.

---

# 18. Mobile First

Toda funcionalidade deve continuar utilizável em dispositivos móveis.

---

# 19. Acessibilidade

Contraste adequado.

Navegação por teclado.

Mensagens claras.

Componentes com foco visível.

Labels obrigatórios.

---

# 20. Teste Final

Antes de aprovar qualquer tela perguntar:

Uma pessoa que nunca utilizou um sistema de RH conseguiria concluir esta tarefa sem ajuda?

Se não conseguir,

a tela ainda não está pronta.
